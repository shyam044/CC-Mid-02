1) SOA Enables Interoperability and Integration Between Different Systems and Technologies

  • SOA is designed to enable interoperability and integration between different systems and technologies
  • SOA stands for Service Oriented Architecture
  • SOA is a way to build software using small parts called Services
  • SOA supports loose coupling, meaning services can work independently
  • Each service does one job and can be reused in other programs
  • Each service communicates with other services by using standard protocols like:
    • HTTP
    • XML
    • SOAP
    • REST
  • There are two major roles in SOA:
    • Service Provider – Creates and publishes services
    • Service Consumer – Finds and uses services

2) Key Challenges to Maintain Stability in Two-Level Resource Architecture

  • Two-level resource architecture uses two controllers:
    • Application-level controller
    • Cloud-level controller
  • Application-level Controller:
    • It manages Local Resources.
    • It allocates the resources based on the demand.
  • Cloud-level Controller:
    • It manages Global Resources.
    • It allocates the resources based on the app request.

Stability in a Two-Level Resource Allocation Architecture:

  • Stability depends upon the coordination between the Application-Level Controller and Cloud-Level Controller.

Coordination Between Levels:

  • Maintains synchronization between the Application-Level Controller and Cloud-Level Controller.

Feedback Mechanism:

  • Continuously monitors the system performance.
  • Uses feedback to automatically adjust the resources.

Dynamic Threshold:

  • Scale Up: Add resources when CPU usage is too high.
  • Scale Down: Remove resources when CPU usage is too low.

3) Best Practices for Optimizing Cost Efficiency and Resource Allocation in the Cloud

  • Resource Check:
    • The system checks if enough resources are available before starting a task.
    • If resources are available, the task is accepted; otherwise, it is rejected.
  • Resource Allocation:
    • Distributes workloads across multiple servers to prevent overloading.
    • Improves performance and avoids bottlenecks.
  • Load Balancing:
    • Load Balancing Algorithm is a technique for distributing the workloads across multiple servers.
    • Types of Load Balancing Algorithms:
      • Static Load Balancing Algorithm
      • Dynamic Load Balancing Algorithm
  • Power Saving:
    • Runs tasks on fewer servers to save power.
    • Puts idle servers in sleep mode.
    • Reduces power consumption and cost.
  • Scheduling:
    • Resources like CPU time and bandwidth are shared among virtual machines using scheduling algorithms.
    • Common Algorithms:
      • Round Robin Algorithm
      • Shortest Job First Algorithm
  • Dynamic Scaling:
    • Scale Up: Add resources when CPU usage is high.
    • Scale Down: Remove resources when CPU usage is low.

4) Block Storage Model

  • Block Storage Model is used for storing and organizing data efficiently.
  • It divides data into fixed-sized units called blocks.
  • Block Storage Model supports read or write operations.
  • Example:
    • Amazon EBS
    • EBS stands for Elastic Block Store

Key Features of Block Storage Model:

  • Independent Blocks:
    • Each block stores and modifies the data independently without affecting other blocks.
  • High Performance:
    • Provides fast access to data with minimal delay.
  • Scalability and Flexibility
  • Data Protection:
    • The data is shared across multiple cloud servers.
    • If one server fails, the data is still safe in the other servers.

Advantages of Block Storage Model:

  • Improves data access speed.
  • Supports custom file system.
  • Scalability.
  • Flexibility.
  • Reliability.

5) Distributed File System

  • DFS stands for Distributed File System
  • In a Distributed File System, files are stored across multiple servers and appear as a single file system to the users.

Key Features of DFS:

i) Transparency:

  • The user can access the files without knowing the storage location.

ii) Resource Sharing:

  • Multiple users share files, hardware, and software applications easily.
  • It reduces the duplication of resources.

iii) High Performance:

  • Provides fast access to the files with minimal delay.

iv) Scalability:

  • DFS increases scalability by adding new servers to the system.

v) Availability & Reliability:

  • Cloud systems store files at different locations around the world.
  • If one server fails, the files are still safe on other servers, which makes it more reliable and highly available.

Example:

  • NFS (Network File System)
  • AFS (Andrew File System)
  • GFS (Google File System)
